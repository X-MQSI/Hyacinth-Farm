# Hyacinth Farm Monitor - 交互式 API 工具

## 功能特点

这是一个功能完整的交互式命令行工具，用于测试和调试 Hyacinth Farm Monitor 的所有 API 接口。

### 主要功能

1. **心跳接口** ❤️
   
   - 发送单次心跳
   - 查询心跳状态
   - 启动心跳循环（每5秒自动发送）

2. **传感器数据接口** 📊
   
   - 发送预设数据（快速测试）
   - 自定义发送数据（手动输入各项参数）
   - 查询最新数据
   - 按时间范围查询
   
   **支持的传感器字段：**
   - 环境温度 (temperature)
   - 土壤湿度 (soil_moisture)
   - 光照强度 (light)
   - 大气压强 (pressure)
   - 电池电压 (voltage)

3. **事件接口** 🔔
   
   - 发送预设事件（浇水、故障、启动等）
   - 自定义发送事件
   - 查询事件列表

4. **调试日志接口** 🐛
   
   - 发送各级别日志（DEBUG/INFO/WARN/ERROR）
   - 自定义日志内容
   - 查询和过滤日志

5. **图片接口** 📷
   
   - 上传图片（指定本地文件路径）
   - 查询图片列表
   - 分页查询

6. **服务状态查询** ℹ️
   
   - 查看服务器运行状态

7. **完整 API 测试** 🧪
   
   - 自动运行所有接口测试
   - 显示测试报告

## 使用方法

### 启动工具

```powershell
# 默认连接本地服务器（需要输入 Session Key）
.\api_tool.ps1

# 指定服务器地址和 Session Key
.\api_tool.ps1 -BaseUrl "http://192.168.1.100:3000" -SessionKey "your-32-character-key-here"

# 连接指定服务器
.\api_tool.ps1 -BaseUrl "http://192.168.1.100:3000"
```

### 操作说明

1. 启动后会显示主菜单（首次运行需要输入 Session Key）
2. 输入数字选择功能
3. 在子菜单中继续选择具体操作
4. 输入 0 返回上级菜单或退出

**注意事项：**

- Session Key 可以从服务器启动日志中获取，或读取服务器根目录的 `.session_key` 文件
- 所有 POST API 请求都会自动附加 Session Key 进行认证
- GET API 请求无需认证（用于数据查询）
- 如果认证失败（401错误），请检查 Session Key 是否正确
- 图片上传和全API测试功能已包含 Session Key 认证
- 网页访问不需要手动输入 Session Key，前端会自动从服务器获取

### 使用示例

#### 示例 1：发送心跳

```
主菜单 → 1 (心跳接口) → 1 (发送心跳)
```

#### 示例 2：自定义传感器数据

```
主菜单 → 2 (传感器数据) → 2 (自定义发送)
然后输入各项参数（留空使用默认值）
```

#### 示例 3：上传图片

```
主菜单 → 5 (图片接口) → 1 (上传图片)
输入图片路径，例如: test.jpg (图片文件放置在同目录下)
```

#### 示例 4：启动心跳循环

```
主菜单 → 1 (心跳接口) → 3 (启动心跳循环)
按 Ctrl+C 停止循环
```

   ```bash

   npm start
   
   # 生产环境（推荐）

   ```
2. 启动工具：`.\api_tool.ps1`
3. 选择功能 7 运行完整测试
4. 查看测试报告确认所有接口正常

## 常见问题

**Q: 提示连接失败？**
A: 检查服务器是否启动，端口是否正确

**Q: 图片上传失败？**
A: 确认文件路径正确，文件格式为 jpg/jpeg/png

**Q: 如何停止心跳循环？**
A: 按 Ctrl+C 键

**Q: 如何修改服务器地址？**
A: 使用参数启动：`.\api_tool.ps1 -BaseUrl "http://your-server:3000"`
